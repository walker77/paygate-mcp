API Key Auth

Cryptographic key generation with pg_ prefix. Keys masked in list endpoints. Admin key never exposed.

Credit Billing

Each tool call costs credits. Set global pricing or per-tool overrides. Per-key spending limits. Integer-only to prevent float exploits.

Rate Limiting

Per-key sliding window rate limits. Set globally or per-tool with independent limits. Concurrent-safe. Default 60 calls/min.

NEW

Per-Tool ACL

Show or hide tools per API key. Enterprise access control — each customer only sees the tools they've paid for.

NEW

Key Expiry

TTL on API keys — auto-expire after N days. Issue trial keys, time-limited access, or rolling credentials with zero cleanup.

NEW

Webhook Events

Real-time notifications for key events: tool calls, low balance, key revocation, and more. HMAC-signed for security.

Usage Metering

Track every call: who, what tool, when, credits charged. Full dashboard via /status. Export JSON & CSV.

Shadow Mode

Log all tool calls without enforcing payment. Perfect for testing and understanding usage patterns before going live.

Two Transports

Wrap local servers via stdio or gate remote servers via Streamable HTTP. SSE streaming, session management, and graceful cleanup.

How it works in practice

1 You wrap your MCP server with PayGate and set a price per tool call
2 You issue API keys to your customers with credits, ACL, and optional expiry
3 Their AI agents call your tools — credits deducted, webhooks fired
4 Connect Stripe — credits auto-top-up when customers pay

New in v0.8.0: Per-tool ACL, per-tool rate limits, key expiry, webhook events, spending limits, refund-on-failure, and config file mode. 232 tests across 12 suites.

Up and running in
60 seconds.

1

Start the gated server

terminal 
npx paygate-mcp wrap \
  --server "npx @modelcontextprotocol/server-filesystem /tmp" \
  --price 2 \
  --rate-limit 30
2

Create an API key with ACL & expiry

terminal 
curl -X POST http://localhost:3402/keys \
  -H "Content-Type: application/json" \
  -H "X-Admin-Key: YOUR_ADMIN_KEY" \
  -d '{"name": "my-client", "credits": 100,
       "allowedTools": ["read_file", "list_files"],
       "expiresIn": "30d"}'
3

Call tools with billing

terminal 
curl -X POST http://localhost:3402/mcp \
  -H "Content-Type: application/json" \
  -H "X-API-Key: CLIENT_KEY" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call",
       "params":{"name":"read_file",
       "arguments":{"path":"/tmp/data.txt"}}}'

Clean, minimal
REST API.

Endpoint Method Auth Description
/mcp POST X-API-Key JSON-RPC 2.0 proxy to wrapped server
/keys POST X-Admin-Key Create key with credits, ACL, expiry, rate limits
/keys GET X-Admin-Key List all keys (masked)
/topup POST X-Admin-Key Add credits to an existing key
/balance GET X-API-Key Client self-service balance check
/keys/revoke POST X-Admin-Key Revoke an API key
/usage GET X-Admin-Key Export usage data (JSON/CSV)
/status GET X-Admin-Key Full dashboard with usage stats
/version GET None Version info from package.json
/ GET None Health check
Free methods: initialize, tools/list, resources/list, prompts/list, and ping pass through without auth or billing. Tools hidden by ACL are filtered from tools/list responses.
